CELESTIAL NEXUS v73 — 26-MEMBER ROSTER FIX

WHY v72 STAYED AT 22
The public GitHub repository still contains the old single-file v67 deployment.
It does not currently contain data/roster.json or the .github workflow, so the
site has nothing automatic to refresh. v72 also allowed a stale SENTRY response
to replace a newer repository snapshot. v73 fixes both problems.

UPLOAD ALL OF THESE PATHS TO THE REPOSITORY ROOT
  index.html
  sw.js
  data/roster.json
  data/roster-overrides.json
  .github/scripts/update_roster.py
  .github/workflows/update-roster.yml

Preserve the data and .github folder names exactly. Keep the repository's
existing manifest.webmanifest, icon-192.png, and icon-512.png files.

RECOMMENDED GITHUB WEB STEPS
1. Extract the ZIP locally.
2. In the repository choose Add file > Upload files.
3. Drag the CONTENTS of the extracted folder into the upload page. Make sure
   both data/ and .github/ appear in the pending file list before committing.
4. Commit directly to main.
5. Open Settings > Actions > General > Workflow permissions and select
   Read and write permissions, then Save.
6. Open Actions > Update Celestial Alliance roster > Run workflow.
7. Confirm data/roster.json shows 26 members and a current generated_at value.
8. Reload the GitHub Page twice or use a hard refresh so the v73 service worker
   replaces the old cached app.

HOW IT STAYS CURRENT
- The updater runs every 6 hours and can also be run manually.
- Official RSI organization member data is the primary source.
- SENTRY is emergency fallback only and cannot overwrite a larger existing
  snapshot with a stale smaller count.
- The website reads the same-origin data/roster.json first; the Sync button
  clears old roster cache and rechecks that file.
- A verified 26-member roster is built into index.html, so a missing data folder
  is clearly labeled as a deployment problem rather than silently showing 22.

CUSTOM DISPLAY LABELS
